The Security API provides several `pgutil` commands (available as `pgutil security`) and HTTP endpoints to create, add, delete, and perform other operations on users, groups and permission. All HTTP requests are made through the following base URL:

:::(info)
 `/api/security/...`
:::

## pgutil Commands

All HTTP endpoints of the The Asset Directories API are available as `pgutil` commands. 

To find the list of commands available in `pgutil`, simply run `pgutil security`. See [Getting started with pgutil](/docs/proget/api/pgutil) to learn more.

The following is a summary of access types and their corresponding requirements for various API keys types and endpoints within this API.

| Access Type | Requirements |
| --- | --- |
| System API Keys | Required for all endpoints

To specify an API Key, use the request header (`X-ApiKey`), querystring (`key`), or `api:¬´api-key¬ª` as the username. See [API Key Usage](/docs/proget/api/apikeys#using-api-keys) to learn more.

:::(Info) (üöÄ Quick Example: Authenticating with curl)

:::

## Data Specifications { #data-specifications }

### SecurityUser Object Properties { #securityuser-object }
`SecurityUser` is a JSON object (see [SecurityUser](https://github.com/Inedo/pgutil/blob/accesso/Inedo.ProGet/SecurityUser.cs)) that correspond to the fields on a User. It's used as input data for both [Create User]() and [Edit User](), and output data for [Show User](), [List Users]() and [List Group Members]().

#### JSON Object Example:

```json
{
    "name": "John Smith",
    "displayname": "jsmith",
    "mail": "jsmith@kramerica.com",
    "password": "password123",
    "groups": [] // refer to SecurityGroups object 
}
```

### SecurityGroup Object Properties { #securitygroup-object }
`SecurityGroup` is a JSON object (see [SecurityGroup](https://github.com/Inedo/pgutil/blob/accesso/Inedo.ProGet/SecurityGroup.cs)) that correspond to the fields on a Group. It's used as input data for [Create Group]() and output data for [List Groups]().

#### JSON Object Example:

```json
{
    "name": "Developers",
    "user": [] // refer to SecurityUsers object
}
```

### SecurityTask Object Properties { #securitytask-object }
`SecurityTask` is a JSON object (see [SecurityTask](https://github.com/Inedo/pgutil/blob/accesso/Inedo.ProGet/SecurityTask.cs)) that correspond to the fields on a Task. It's used as input data for [Create Task]() and output data for [List Tasks]().

#### JSON Object Example:

```json
{
    "name": "Example",
    "description": "This is an example",
    "feedscoped": ???,
    "password": "password123",
    "attributes": [] //refer to attributes object
}
```

### SecurityPermission Object Properties { #securitypermission-object }
`SecurityPermission` is a JSON object (see [SecurityPermission](https://github.com/Inedo/pgutil/blob/accesso/Inedo.ProGet/SecurityPermission.cs)) that correspond to the fields on a Permission. It's used as input data for [Add Permission]() and output data for [List Permissions]().

#### JSON Object Example:

```json
{
    id
    user
    group
    task 
    feed 
    feedgroup
    deny 
}
```
