The Security API provides several `pgutil` commands (available as `pgutil security`) and HTTP endpoints to create, add, delete, and perform other operations on Users, Groups, Tasks and Permissions. All HTTP requests are made through the following base URL:

:::(info)
 `/api/security/...`
:::

## pgutil Commands

All HTTP endpoints of the Security API are available as `pgutil` commands. 

To find the list of commands available in `pgutil`, simply run `pgutil security --help`. See [Getting started with pgutil](/docs/proget/api/pgutil) to learn more.

## Authenticating to Security API { #authentication }

The following is a summary of access types and their corresponding requirements for various API keys types and endpoints within this API.

| Access Type | Requirements |
| --- | --- |
| System API Keys | Required for all endpoints

To specify an API Key, use the request header (`X-ApiKey`), querystring (`key`), or `api:¬´api-key¬ª` as the username. See [API Key Usage](/docs/proget/api/apikeys#using-api-keys) to learn more.

:::(Info) (üöÄ Quick Example: Authenticating with curl)

:::

## Data Specifications { #data-specifications }

### SecurityUser Object Properties { #securityuser-object }
`SecurityUser` is a JSON object (see [SecurityUser.cs](https://github.com/Inedo/pgutil/blob/accesso/Inedo.ProGet/SecurityUser.cs)) that correspond to the fields on a User. It's used as input data for both [Create User]() and [Edit User](), and output data for [Show User](), [List Users]() and [List Group Members]().

#### JSON Object Example:

```json
{
    "name": "John Smith",
    "displayName": "jsmith",
    "mail": "jsmith@kramerica.com",
    "password": "password123", // this will be hidden in output
    "groups": ["Developers"] // refer to SecurityGroups object 
}
```

### SecurityGroup Object Properties { #securitygroup-object }
`SecurityGroup` is a JSON object (see [SecurityGroup.cs](https://github.com/Inedo/pgutil/blob/accesso/Inedo.ProGet/SecurityGroup.cs)) that correspond to the fields on a Group. It's used as input data for [Create Group]() and output data for [List Groups]().

#### JSON Object Example:

```json
{
    "name": "Developers",
    "user": [] // refer to SecurityUsers object
}
```

### SecurityTask Object Properties { #securitytask-object }
`SecurityTask` is a JSON object (see [SecurityTask.cs](https://github.com/Inedo/pgutil/blob/accesso/Inedo.ProGet/SecurityTask.cs)) that correspond to the fields on a Task. It's used as input data for [Create Task]() and output data for [List Tasks]().

#### JSON Object Example:

```json
{
    "name": "Basic Access",
    "description": "This allows basic access",
    "feedScoped": true,
    "attributes":    
     [
      "Feeds_ViewFeed",
      "Feeds_DownloadPackage",
      "Feeds_PullPackage"
      ]
}
```
### SecurityTaskAttribute Object Properties { #securitytaskattribute-object }
`SecurityTaskAttribute` is a JSON object (see [SecurityTaskAttribute.cs](https://github.com/Inedo/pgutil/blob/accesso/Inedo.ProGet/SecurityTaskAttribute.cs)) that correspond to the fields on a Task Attribute. It's used as output data for [List Task Attributes](/docs/proget/api/security/tasks/list-attributes).

#### JSON Object Example:

```json
{
    "name": "Feeds_ViewFeed",
    "feedScoped": true
}
```

### SecurityPermission Object Properties { #securitypermission-object }
`SecurityPermission` is a JSON object (see [SecurityPermission.cs](https://github.com/Inedo/pgutil/blob/accesso/Inedo.ProGet/SecurityPermission.cs)) that correspond to the fields on a Permission. It's used as input data for [Add Permission]() and output data for [List Permissions]().

#### JSON Object Example:

```json
[
  {
    "id": 1,
    "group": "Administrators",
    "task": "Administer",
    "deny": false
  },
  {
    "id": 3,
    "user": "David Smith",
    "task": "View & Download Packages",
    "feed": "unapproved-nuget",
    "deny": true
  },
  {
    "id": 4,
    "group": "developers",
    "task": "View & Download Packages",
    "feedGroup": "production-packages",
    "deny": false,
  },
  {...}
]
```
